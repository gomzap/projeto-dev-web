# G-Tech | Hardware & Security

## 📌 Visão Geral do Projeto
O **G-Tech** é um projeto de website voltado para comércio eletrônico, desenvolvido como requisito parcial para a disciplina de Desenvolvimento Web. O sistema está sendo construído em três etapas incrementais: HTML5/CSS, JavaScript e PHP com Banco de Dados.

## 🎯 Definições do Projeto
* **Tema do site:** Mini-ecommerce especializado em hardware de alta performance (focado em jogos competitivos como CS2 e Valorant) e soluções corporativas de Segurança da Informação (como chaves FIDO2).
* **Público-alvo:** Gamers, entusiastas de tecnologia, profissionais e estudantes de cibersegurança que buscam equipamentos de ponta.

## 🗺️ Mapa de Páginas
A navegação foi estruturada de forma simples e direta, contemplando os requisitos da Etapa 1:

1. `/index.html`: **Página Principal (Home).** Apresentação institucional e catálogo de produtos em destaque organizados em formato de vitrine estruturada.
2. `/contato.html`: **Página de Atendimento.** Formulário completo para contato comercial e orçamentos, utilizando validações nativas do HTML5 (tipos email, tel, number, e atributos como required, min, max e pattern).
3. `/css/estilos.css`: **Folha de Estilos Externa.** Responsável por toda a identidade visual, tipografia, cores e layout.

## 🚀 Instalação e Execução
Para esta **Etapa 1 (HTML5 + CSS)**, o projeto é totalmente estático no lado do cliente (client-side) e não requer a instalação de servidores locais. 

**Roteiro para testar a aplicação:**
1. Faça o download ou clone o repositório do projeto.
2. Extraia os arquivos mantendo a estrutura original (o arquivo `estilos.css` deve permanecer dentro da pasta `css`).
3. Dê um duplo clique no arquivo `index.html` ou abra-o no navegador de sua preferência (Google Chrome, Firefox, Edge).
4. Navegue pelo menu superior para testar o roteamento entre as páginas e interaja com o formulário para testar os bloqueios de entradas inválidas do HTML5.

## 🛠️ Tecnologias e Padrões Utilizados
* **HTML5:** Estrutura construída com tags semânticas (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`) para melhor indexação, organização e acessibilidade básica.
* **CSS3:** Estilização desenvolvida em folha externa, focando em legibilidade e organização.
* **Boas Práticas:** Separação de responsabilidades (pastas organizadas), código identado e clareza de navegação.

## 📅 Próximas Etapas do Desenvolvimento
O projeto continuará evoluindo conforme o cronograma da disciplina:
* **Etapa 2:** Implementação de validações dinâmicas e interatividade avançada no client-side utilizando **JavaScript**.
* **Etapa 3:** Criação do back-end em **PHP** e integração com banco de dados para gerenciar o fluxo CRUD completo.
